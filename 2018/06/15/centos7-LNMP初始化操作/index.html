centos7+LNMP初始化操作

Jun 15, 2018

环境 cnetos7+ nginx + mysql + php

LNMP安装

安装步骤详见LNMP安装

  1. 安装完成后，发现直接用公网ip无法访问默认页面，如果遇到这种情况，直接去查看阿里云后台实例的安全组开放下80端口即可。

开放3306端口

Mysql为了安全性，在默认情况下用户只允许在本地登录，可是在有此情况下，还是需要使用用户进行远程连接，因此为了使其可以远程需要进行如下操作：
一、允许root用户在任何地方进行远程登录，并具有所有库任何操作权限，具体操作如下：
在本机先使用root用户登录mysql：

mysql -u root -p”youpassword”

进行授权操作：
mysql>GRANT ALL PRIVILEGES ON . TO ‘root’@’%’ IDENTIFIED BY ‘youpassword’ WITH GRANT OPTION;

重载授权表：
FLUSH PRIVILEGES;

退出mysql数据库：
exit

用navicat远程链接数据库发现出现如图错误：

CentOS 7.0默认使用的是firewall作为防火墙，这里改为iptables防火墙。
解决方法：

  1. 关闭firewall

  2. 安装iptables防火墙

  3. 启动设置防火墙

  4. 查看防火墙状态

  5. 编辑防火墙，增加端口

  6. 重启配置，重启系统

这里需要注意的是：如果阿里云实例安全组没有开放3306端口也连接不上的，开放端口即可
